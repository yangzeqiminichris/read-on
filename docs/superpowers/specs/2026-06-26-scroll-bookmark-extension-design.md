# ReadOn · 设计文档

> **ReadOn**（续读）——在网页里标一个位置，下次直接续着读下去。

**日期**：2026-06-26
**状态**：已确认，待进入实现计划

## 1. 背景与目标

学习在线长文档（如 LangChain 官方 doc）时，常常需要从上次中断的位置继续阅读。Chrome 原生书签只记到"哪个网页"，不记"网页的哪个位置"；现有插件各有短板：

- **Mark Scroll Positions (MSP)**：功能最全（一页多 mark、可写笔记、page/screen ratio 两种跳转策略），但偏"网页笔记本"。缺点：mark **不能更新位置**（只能删旧建新）；查看 All Marks 必须**跳到新标签页**；**无分享**功能。
- **Page Position Bookmark**：最轻量，但**只能在对应页面里看到该页的位置标签**，无法跨页面知道"上次看的是哪个章节"。
- **Scroll In**：有 **update** 功能（适合长页面随时更新），但**不能加笔记**、且查看 marks 也要跳新页面。

**目标**：做一个"既要又要"的本地 Chrome 扩展，以**续读（轻量）为主**，同时支持**笔记**，取三者之长、补其之短。

### 服务两类用户

1. **轻量党**：一页只要一个 mark，记住"上次看到这"，下次反复 **update** 位置继续看。
2. **笔记党**：一页可建多个 mark、在不同位置写心得，并可**导出分享**给别人。

## 2. 技术栈与约束

- **原生 JS + MV3**，**无构建步骤**（写完直接"加载已解压的扩展"）。
- **浏览器 API 集中到单一 wrapper 文件**（`storage.js` 等），所有 `chrome.*` 调用收敛于此——这是唯一的前瞻性投资，为日后移植 Firefox（`webextension-polyfill` + 多 manifest）只改一处。Edge 本就是 Chromium，几乎原样可上。
- 构建流水线**后期再搭**（等 Chrome 反响好），因代码模块化 + API 收敛，届时改动很小。
- 分享采用**导出/导入 JSON 文件**，**纯本地零服务器**。
- CSP 注意：MV3 禁止远程代码与 `eval`，本方案无外部依赖、无内联脚本，天然合规。

## 3. 整体架构

四个解耦组件：

1. **Content Script**：注入网页。负责"读位置"（抓滚动快照 + 文字锚点）与"跳位置"（执行定位算法滚动）。
2. **Popup**：主界面。默认显示**当前页面的 marks**，顶部可切到弹窗内的 **All Marks** 视图。增删改、update、跳转、导入导出入口皆在此。
3. **存储层 (storage wrapper)**：唯一接触 `chrome.storage.local` 的模块，所有 mark 数据存本地。未来移植的唯一改动点。
4. **完整管理页（Options 全屏页）**：仅"复杂需求"（批量管理、搜索、编辑长笔记、整体导出）才进入。是唯一会打开新标签页的地方。

**数据流**：Popup 经存储层读写数据；需要抓当前位置或执行跳转时，Popup ↔ Content Script 通过消息通信。

## 4. 数据模型

```text
mark: {
  id,                       // 唯一 id
  name,                     // 不可改；默认 "Mark #N"
  pageKey,                  // hostname + pathname（去 hash），用于 All Marks 归类
  pageURL, pageTitle,       // 展示 / 跨页面跳转用
  note,                     // 笔记（可随时改；轻量党留空）
  createdAt, updatedAt,     // update 位置时只改 updatedAt

  // —— 定位信息 ——
  scrollPosition,           // 保存时的像素滚动量（同 MSP）
  viewportHeight,
  contentHeight,
  strategy,                 // 'page-ratio' | 'viewport-ratio'，可按站点设
  anchorText,               // 保存时视口顶部元素的 textContent 片段（消歧 + 精确定位）
  scrollContainerSelector,  // 可选，给滚内部容器的网页用（同 MSP）
}

pageData: {
  pageKey,
  marks: [...],
  nextSeq,                  // 每页单调递增计数器；新建时 name = "Mark #" + nextSeq，然后 nextSeq++
}
```

### 命名规则

- `name` 新建时可自定义，**一旦确认即不可改**（与 MSP 一致）。
- 默认名 `Mark #N` 递增，**单调递增、永不复用**：删除后编号可有空缺（如 #2、#3 而无 #1），但绝不撞名——名字不可改，唯一性是底线。

## 5. 定位算法（比例 + 文字锚点）

MSP 实为**纯比例定位**（只存 `scrollPosition/viewportHeight/contentHeight`，无文字锚点），故动态加载/改版后会漂移。本方案在其基础上叠加文字锚点，两个信号互补：

| 信号 | 单独的弱点 | 擅长 |
|---|---|---|
| 比例（同 MSP） | 内容变化后漂移 | 给出**大概区域** |
| 文字锚点 | 重复文字撞车 | 小范围内**精确定位某一行** |

**保存**：存 MSP 那套数字 **+** 当时视口顶部元素的一段 `textContent`（取整段，越长越不易撞）。

**两种比例策略**（同 MSP）：
- `page-ratio`：`scrollPosition / (contentHeight - viewportHeight)`，0~1 占整页比例。
- `viewport-ratio`（screen ratio）：`scrollPosition / viewportHeight`，"往下几屏"。

**跳转**：
1. 先用比例算出大概目标位置（同 MSP）。
2. 在页面查找 `anchorText` 的所有出现位置。
3. 若多处（重复），选**离第 1 步比例位置最近**的那处，精确滚过去。
4. 若找不到（页面大改），**退回纯比例**——最坏不比 MSP 差。

锚点针对"视口顶部最近的**元素**"，而非在 HTML 源码里搜字符串，更稳。

## 6. 核心交互流程

**① 新建 mark（`Mark` 按钮）**
点 `Mark` → 立即创建 `Mark #N`，就地**只编辑名字**（默认填好、可改），确认/失焦后锁定。不在创建时填笔记。创建时 content script 抓快照，`nextSeq++`。

**② 笔记（延后填）**
每个 mark 行末尾有展开箭头 `⌄`。展开后显示：**创建日期** + **删除 🗑** + **记笔记 📝**。点记笔记 → 弹出 textarea 写/改笔记（笔记可随时改）。轻量党永不展开。

**③ update（更新位置）**
与 ▶ 并排的独立按钮 `⟳`，hover 提示 "Update this mark's position"。点击 → 重抓快照覆盖**定位字段 + `updatedAt`**，名字/笔记不动。给轻提示（toast）。

**④ 跳转（▶ 按钮）**
- 本页：直接执行第 5 节定位算法滚动。
- 跨页面（从 All Marks 点）：先把当前标签页导航到 `pageURL`，**等加载完且内容高度稳定**后再滚动。

**⑤ All Marks（弹窗内视图）**
弹窗顶部切换，列出**所有页面**的 marks，**按页面分组**（页面标题 + URL），每条显示名字、滚动百分比、笔记预览。点任意条 = 走④跨页面跳转。底部"打开完整管理页"入口（唯一开新标签页处）。

**⑥ 导出 / 导入（分享）**
- **导出**：数据导成 JSON 文件下载（管理页可选"全部"或"某页面"）。
- **导入**：选文件 → 合并。规则：**按 mark `id` 去重**，已存在跳过，新的追加到对应 `pageKey`。别人的笔记与自己的并存、不互相覆盖。

### Mark 行布局

```
○  Mark #1              [▶]  [⟳]  [⌄]
   93% scrolled        跳转 更新 展开
   └─展开后─ 2026年6月26日        [🗑] [📝记笔记]
            （点📝）→ textarea
```

## 7. 边界情况处理

| 情况 | 处理 |
|---|---|
| 锚点文字找不到（页面大改） | 退回纯比例定位 |
| 锚点文字多处（重复） | 选离比例目标位置最近的那处 |
| 跨页面跳转、页面仍在加载/懒加载 | 导航后等加载完 + **等内容高度稳定**（短轮询/重试）再滚动 |
| 网页滚的是内部容器 | 用 `scrollContainerSelector` |
| 页面变短（内容被删） | 目标位置 clamp 到可滚动范围 |
| 受限页面（`chrome://`、商店页、PDF 等） | `Mark` 按钮置灰，提示不支持 |
| 导入文件损坏/格式错 | 校验 schema，失败明确报错，不污染已有数据 |

## 8. 测试策略

配合无构建、原生 JS：

- **纯函数单元测试**（Node 自带 `node:test`，无需浏览器/构建）：比例换算、重复锚点"选最近"消歧、导入合并去重、`nextSeq` 递增。把定位算法与合并逻辑抽成纯函数模块独立覆盖——项目质量关键。
- **存储层**：mock `chrome.storage` 测读写/合并。
- **content script 的 DOM 抓取与滚动**：手动验收清单，拿 LangChain 文档当样板：
  1. 轻量党续读流程（建 1 mark → update → 跳回）
  2. 笔记党多 mark + 笔记流程
  3. 跨页面跳转（All Marks → 点入 → 定位）
  4. **抗漂移**：折叠一段内容后再跳，验证文字锚点纠正
  5. 重复文字页面定位正确
  6. 导出 → 导入往返数据一致

## 9. 明确不做（YAGNI）

- 在线分享 / 后端服务器（用导出导入文件替代，后期视反响再升级）。
- 构建流水线 / 框架（后期需要时再加，改动小）。
- 跨设备云同步（先纯本地 `chrome.storage.local`）。
- 名字的二次编辑（设计上即不可改）。
